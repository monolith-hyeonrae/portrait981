# 프랜차이즈와 데이터 해자

> 테마파크 어트랙션은 person-conditioned face diversity 연구를 위한 자연 실험실이다.

---

## 왜 이 데이터가 희소한가

person-conditioned diversity 모델에 필요한 데이터의 핵심 조건:
**같은 사람이 자연스럽게 여러 표정을 짓는 상황을 반복적으로 관찰.**

이 조건을 충족하는 데이터 소스가 세상에 거의 없다.

| 데이터 소스 | 같은 사람 반복 | 자연스러운 표정 | 다양한 감정 맥락 | 연속 촬영 |
|------------|:------------:|:-------------:|:--------------:|:--------:|
| 연구실 촬영 | 1회 | 연기 | 지시에 의존 | 짧음 |
| 공개 데이터셋 (AffectNet 등) | 대부분 1장 | 혼재 | 제한적 | 스냅샷 |
| CCTV/거리 | 재관찰 불가 | 자연 | 단일 | 제한적 |
| 영상 통화 | 가능 | neutral 위주 | 대화에 제한 | 가능 |
| **테마파크 어트랙션** | **연간 패스** | **진짜 감정** | **어트랙션별** | **1~2분** |

## 테마파크의 구조적 이점

어트랙션 탑승 1~2분 동안 자연 발생하는 표정 전환:

- 출발 전: 긴장/기대
- 탑승 중: 놀람/흥분/공포
- 하이라이트: 최고조 표정
- 종료 후: 안도/웃음

**한 번의 탑승에서 2~3가지 자연스러운 표정 전환.**
여러 어트랙션을 타면 감정 맥락이 더 다양해진다.

## 프랜차이즈 시너지

| 규모 | 데이터 효과 |
|------|-----------|
| 단일 지점 | N명 × M개 어트랙션 |
| 다중 지점 | population prior 정교화 + 어트랙션 다양화 + 교차 방문 |
| 글로벌 | 지역/문화별 표현 차이 → 범용 모델 |

티켓 매출은 선형이지만, 데이터 자산 가치는 **초선형** (네트워크 효과 + 복리 축적).

## 경쟁 구도

```
구글:    기술 ✓  인프라 ✗  관점 ✗  → 데이터 구조가 없음
디즈니:  기술 △  인프라 ✓  관점 ✗  → 할 수 있지만 하지 않음
981파크: 기술 ✓  인프라 ✓  관점 ✓  → 유일하게 실행 중
```

- **구글**: YouTube에 영상은 수십억 개이지만, "같은 사람의 반복적 자연 표정 + identity 보장"은 없음. 기술력으로 데이터 구조를 만들 수 없음 — 테마파크를 운영해야 함.
- **디즈니**: MagicBand + PhotoPass로 인프라는 있지만, "사진 판매" 사업으로 고정. 분석 파이프라인 없음, per-person distribution 개념 없음.
- **981파크**: "사진을 찍는 것"이 아니라 "사람을 이해하는 것"이라는 관점. visual* 파이프라인이 매 촬영을 데이터 자산으로 변환.

핵심: **데이터의 양이 아니라 구조가 결정적이고, 그 구조는 사업 형태에서 나온다.**

## 데이터 플라이휠

```
테마파크 운영 → 영상 자동 수집
  → visual* 14 frozen models → 49D signals → pseudo-labels
  → per-member distribution 축적
  → 모델 개선 → 서비스 가치 향상
  → 프랜차이즈 매력 → 지점 증가 → 데이터 증가 (반복)
```

## 고객 가치 전환

기존: "오늘 찍은 사진 사세요" (단발성)

person-conditioned distribution 시대:
- "당신의 3년간 표정 포트폴리오입니다"
- "지난번에 없던 이런 표정이 새로 수집되었어요"
- "당신만의 표현 스펙트럼이 이만큼 채워졌어요"

→ 방문할수록 포트폴리오가 풍부해지는 **누적 자산**
→ 일회성 상품에서 장기 구독 가치로 전환
